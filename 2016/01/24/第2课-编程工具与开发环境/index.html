第一节 C语言的开发环境及DIY自己的开发环境

常见的C语言开发环境：

  1. 开发环境的组成：
    • 编辑环境；
    • 编译环境；
    • 调试环境；
    • 版本管理器。
  2. C语言常见的开发环境：
    • TurBlo C;
    • VC6;
    • BC;
    • VS系列；
    • CODEBLOCKS。

CodeBlocks的安装和配置————配置自己的C开发环境：

  1. codeBlocks的安装：下载安装包后，直接下一步完成（略过）

  2. 编译器的配置：打开setting ——> compiler ——> GNU GCC compiler ——> ToolChain executables ——> 选择mingw64（事先下载好）文件夹 ——> 在C compiler选择编译器gcc.exe（在mingw64/bin下）——> 选择动态库链接g++.exe(在mingw64/bin下) ——> 选择静态库链接ar.exe（在mingw64/bin下）——> debug选择GDB/CDB debuger:Default ——> resource compiler选择winders.exe(在mingw64/bin下) ——> 点击OK

  3. 调试环境的配置：下载gdb ——> 解压后，复制gdb.exe和libiconv-2.dll到mingw64/bin下 ——>打开setting ——> debuger ——>选中前三项 ——> 点击左侧default ——> 选择Executable path(刚刚复制到bin下的gdb.exe) ——> debuger type 选择 DGB ——> 选择中前四项 ——> 点击OK

第二节 一起熟悉下CodeBlocks

用CB创建第一个C语言工程 

#include <stdio.h>
#include <stdlib.h>

int main()
{
    printf("Hello World!\n");
    return 0;
}

stdio.h 和stdlib.h 是两个C系统库的头文件，系统的头文件写在<>里，自定义的头文件写咋“”里，int main(){}是系统的入口函数，每个C语言项目都有且仅有一个mian函数。int是main函数的返回值类型，每个函数都有返回值，man函数的返回值是int类型的。main函数的()里可以有一些参数，也可以没有。{}包含着main函数的作用域，作用域里的printf(“Hello World!\n”);是打印Hello World!\n的指令，return 0是返回值。
