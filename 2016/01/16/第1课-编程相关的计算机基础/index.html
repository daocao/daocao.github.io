第一节 从计算机的组成谈程序的运行

计算机的基本组成

  • CPU（计算机的大脑，处理所有的计算、逻辑运算）
  • 主板（计算机对外的接口）
  • 内存（RAM）
  • 硬盘(数据存储)
  • 输入设备（鼠标、键盘）
  • 输出设备（显示器、影响）

一个程序是如何运行的

  1. 程序猿编写源代码；
  2. 源代码存储在硬盘中；
  3. 通过输入设备触发程序启动，这时程序加载到了内存中；
  4. CPU读取并执行内存中的程序的指令、数据;
  5. CPU调用输出设备的接口，输出人类可读的信息。

集成开发环境的组成：

  • 编辑工具
  • 编译环境
  • 调试工具

一张图看懂代码道程序的过程：

  1. 源代码Source Code （源文件，即Hello.c）
  2. 头文件Header Files (头文件，即stdio.h)
  3. .c文件 和 .h文件，经过gcc的Preprocess(预处理)生成hello.i文件。（gcc按照移动的规则，把.c文件和.h文件组合在一起，生成.i文件）
  4. hello.i文件通过gcc的编译器Compilation(编译)生成hello.a文件，hell.a是汇编代码。
  5. hello.a通过汇编器（Assembly）生成二进制文件hello.o文件
  6. hello.o文件通过链接器Linking，与链接库相链接，最后生成，和执行文件a.out(Linux中.out文件，win中.exe文件)

第四节 程序和内存空间模型

内存空间的基础知识：

  1. 物理内存：PC里的内存条、某些单片机里自带的RAM
  2. 内存空间模型：程序能看到的内存，程序访问物理内存时，要通过内存空间模型。在PC上内存空间模型是物理内存的映射，生成了一个映射关系的表。
  3. 内存空间基本单位：字节，1字节 = 8bit，1bit记录一个二级制码。
  4. 内存空间要素：
    • 空间大小：32位系统最大使用内存计算：0xFFFF
    • 空间地址：程序中的指针

Tip:bit：位，byte：字节，1byte = 8bit

一张图认识运行中程序的内存空间模型：

  1. 程序运行前，是存储在硬盘里的（ROM）；
  2. 程序运行后，程序本身的代码（包括指令、数据）被编译成了二进制代码，放入内存（RAM）中：
    • 指令、数据放入①中（程序代码区）；
    • 常量被放入②中，常量区；
    • 未初始化的全局变量和静态变量放入③中；
    • 已初始化的全局变量和静态变量放入④中（②③④属于静态数据区）；
    • 通过malock申请的内存被放入⑤中，堆区；
    • 局部变量被放入⑥中，栈区；

第五节 字符编码基础——谈谈asc码

字符在计算机中是如何存储的：

字符在计算机中，是按一定的编码规则用二进制来存储的。

  1. 英文字符的存储：英文字符是按ASC码存储的。ASC码中包含了英文字母大小写、数字、常用/特殊符号，每个ASC码对应了一个数（16进制、10进制、2进制），这些数以二进制的方式存储在计算机里面。
  2. 中文字符的存储：中文字符是按Unicode码存储的。中文字符通过Unicode编码成二进制存储在计算机里面。

计算机编码基础————ASC码：

  1. 什么是ASC码表：用7位字符集表示标准的美式键盘上的每个符号
  2. 如何查看ASC码表：

Tip:ASC码使用技巧：

  • 数字字符到数字的转换；
    • 字符转数字：数字字符对应的ASC码十进制-48，十六进制-0x30。
  • 大小写字母的转换。
    • 大写字母转小写字母：字母对应的ASC码十进制+32，十六进制+0x20。
